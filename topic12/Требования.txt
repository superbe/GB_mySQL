Общая постановка

Сверхзадача собрать в одном месте значимые памятники русской культры.

Помимо самих памятников сайт должен содержать какие-то справочные материалы, 
библиографии, авторитетные мнения и прочее.

Сайт решается в виде небольшой социальной сети.

Участники сети делятся на группы по ролям: администраторы, авторы, модераторы, 
собственники, редакторы, эксперты читатели, гости. Каждый участник может 
обладать несколькими ролями одновременно. В зависимости от ролей приписываемых 
участнику он наделяется соответствующими правами.

Права по каждой роли [в работе].

Основная сущность - памятник культуры. Запись: название, аннотация, тексты.

Каждый памятник может иметь несколько изданий.

Каждый памятник и его издания привязаны ко времени и месту создания.

Каждый памятник и его издания помимо текстов могут иметь изображения 
с отсканированными страницами.

Помимо этого к каждому памятнику или его изданию должны быть привязаны 
сопутствующие материалы: обзоры статьи и другие публикации, которые тоже 
должны иметь библиографическую запись.

Для каждый памятника на главной странице памятника должна отображаться
карта с временным треком и таймлайном. При проигрывании таймлайна на карте
в динамике должен отображаться путь жизненного цикла памятника вплоть 
до гибели. Так же вместе с памятником должны отображаются треки изданий
памятника.

Каждый памятник, его издания или публикации о нем являются документами.

Каждый документ должен иметь собственника и автора.

Каждыйдокумент должен иметь библиографическую запись и привязку к известным 
классификаторам.

Используемые классификаторы ББК, УДК, СК (свой классификатор). Все 
классификаторы строятся по фасетному принципу.

Классификаторы отображаются в виде связанного нерегулярного иерархического
дерева, к каждому разделу которого должны быть привязаны каталожные карточки 
документов.

Каталожные карточки должны содержать библиографическую запись документа, 
номер документа, полочный шифр, ключевые слова и т.д.

Каталожные карточки отображаются в виде настраиваемого списка.

При клике на карточку должен осуществляться переход к документу.

На каждый документ должен формироваться поисковый образ документа (ПОД).
Поисковый образ формируется следующим образом: из документа выделяется словник,
список слов и словосочетаний (понятий), которые с помощью стеммера Портера 
(позже перестроить на стеммер по Зализняку) приводятся к канонической форме 
(лексическая единица), каждая лексическая единица в полученном списке 
оценивается в отношении документа и всей библиотеки, если оценка 
удовлетворительная (значение оценки превышает некоторое критическое значение 
для данной лексической единицы), то она включается в ПОД. Между лексическими 
единицами ПОД формируются отношения обшее-частное в виде указателя документа.

Из указателя документа формируется тезаурус, который должен быть основой для СК.

На основе ПОД строится индекс: все лексические единицы всех ПОД собираются в 
единый список, каждый элемент которого привязываются к конкретному документу.

Так же необходимо создать конкорданс - перечень всех слов с отсылками 
к конкретному месту в документе.

ПОД, указатель, индекс, конкорданс, тезаурус и классификаторы образуют 
справочно-поисковый аппарат.

Поиск должен быть реализован следующим образом: пользователь вводит любую фразу 
в свободном виде, алгоритм с помощью стеммера Портера получает лексические 
единицы в канонической форме, удаляет стоп-слова, далее по индексу формирует
список найденных документов отсортированный по релевантности.

Поиск модет быть: релевантным, пертинентным. В первом случае возвращается 
релевантный список найденных документов. Во втором случае возвращается 
тематический список документов, т.е. релевантный список дополняется документами
из разделов классификатора к котором привязаны найденные документы.

Классификатор, каталожная карточка, тезаурус и т.д. должны оформляться согласно
СИБИД.

Частная постановка на альбом

Изображения сканов страниц документа формируются в альбом. К каждому такому 
изображению привязывается размеченный текст. При переходе к просмотру альбома
должен отображаться список миниатюр сканов страниц при нажатии на одно 
из изображений мы переходим на страницу просмотра этого изображения в увеличеном 
виде, но при этом вверху для других страниц текущего документа должны 
отображаться миниатюры сканов страниц, а внизу миниатюры все запараллеленые 
изображения из других документов. При нажатии на изображение из таблицы 
параллельных мест должен происходить переход в альбом этого документа. 
Для пролистывание миниатюр использовать механизм "карусели".

Все переходы откуда-куда сохранять в журнале состояния. 

Загрузить текущий альбом, далее уже в режиме "редактора" выбрать другие 
альбомы для связывания. Выбрать страницу для связывания в текущем альбоме. 
ДВ связываемом альбоме пролистать до нужной страницы, при клике на эту страницу, 
выбрать для связывания (отображать в увеличенном виде, при повторном клике 
уменьшить до исходного размер миниатюры, если это нужная страница, то 
пользователь должен поставить галочку "Связать"). Перейти к следующей странице.
